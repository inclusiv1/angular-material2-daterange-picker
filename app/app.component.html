This is a POC implementation of the "RANGE" feature for material datepicker using inheritance only!

Net changes are approx 75 LOC, all other code is boilerplate due to template inheritance and component metadata.

The concept is simple, use 2 inputs instead of complicated masking, the input logic for date is already packed within material so it's basically very simple.

The component is in SINGLE mode by default, on if we register a special (adiitional) input element to it it will enter "RANGE" mode. the original value used for SINGLE mode is now the START DATE value in RANGE MODE, and an additional range end value is used to mark the value of the end of the range.

Another issue for discussion is the 2nd input, currently it's metadata attributes are ignored. Things like min/max/filter... these are taken from the 1st input...

As allways, naming of the properties (@Input) are up to discussion :), I admint the names might change
